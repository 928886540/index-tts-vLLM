# TAVO 播放器优化计划 2

记录时间：2026-05-29

目标：先修体验 bug，再调生成质量。

执行约束：
- 2026-05-29 变更开始时用户正在使用服务，禁止跑真实 TTS 压测。
- 用户随后确认“不会用 TTS 了”，允许完整改动和必要验证。
- 默认验证顺序：`node --check static\tavo.js`、`python -m py_compile indextts2_api.py`、`node Leon_api\dev_tools\test_tavo_widget_playwright.js`。
- 固定测试页只检查挂载、懒加载、设置面板、音色选择器；不要新增浏览器或在仓库安装 `node_modules`。

## 必须先做

1. 角色配置跨消息同步
   - 问题：同角色上一条消息改完设置，往前一条消息生成没有生效。
   - 方向：角色级配置必须走 `tavo.set(..., "character")`，每个消息播放器挂载/生成前都重新拉当前角色配置；同一角色的所有消息共享 `defaultVoice`、`roleVoiceList`、质量档、离线音频设置。

2. 播放器续播和后台控制
   - 后台媒体控制不做上一首/下一首，改成后退 10 秒/前进 10 秒。
   - 流式暂停后续播不能从头开始；元素音频用 `currentTime`，WebAudio 链路记录 `lastWebAudioSec`，续播从最近时间点继续或至少不重置 UI 状态。

3. 离线音频闭环
   - 有离线先播离线。
   - 没离线先在线播，不阻塞播放；在线播放成功/落盘后再保存 IndexedDB。
   - 不允许打开首页或恢复历史时批量同步所有消息音频。
   - 若保存失败，下一次用户点播放同一张卡时补偿保存。

4. 首页历史提示
   - 懒加载也要显示历史音频条数。
   - 没有音频时提示“点播放生成音频”，不要提示“加载离线音频”。

## UI/交互

5. 设置页底部对齐
   - 当前设置页又跑到上面；恢复底部抽屉式对齐，移动端优先。

6. 右上角设置按钮和计数器
   - 坐标/计数器与设置按钮不协调、没对齐。
   - 统一高度、间距、圆角、视觉层级。

7. 旁白 icon
   - 当前旁白 icon 太丑。
   - 换成更轻的 narrator 图标；不要占歌词高度。

8. 歌词抖动
   - 固定歌词区高度已经做了，但当前行切换仍抖。
   - 去掉 current 行字号/高度变化，改用颜色、背景、轻微 glow 或左侧标记表示当前行。
   - 滚动居中加节流，避免每 150ms 大幅改 `scrollTop`。

9. 音色映射默认行
   - 默认三行：旁白、用户、角色。
   - 角色行不锁定；旁白/用户可保留常驻，但不要阻止用户配置音色。
   - 没配置音色映射时要提醒，不要静默用宝岛甜心。

10. 设置分组
   - 质量切换移到 LLM 设置外面，单音色和多音色都能配置。
   - “保存离线音频”新建独立播放/离线分组，不和播放模式黏在一起。

## 生成质量

11. 默认质量档
   - 当前只有“质量”档听起来正常。
   - 默认改为 `expressive`，并把 UI 文案改清楚：质量优先。
   - 其它档保留但不作为默认。

12. 参数略提高
   - 在质量档基础上小幅提高稳定性参数，避免部分音色生成不可用。
   - 先改默认值，不做真实压测，等用户空闲后再跑 RTF/听感。

13. 情绪仍机械
   - 多音色模式要更合理地使用 style / style_alpha / emo_vec。
   - 旁白情绪压低，角色台词允许更明显的情绪参考音频。

14. 首块已起播但没声音
   - 控制台“首块已起播”过早。
   - WebAudio 只有实际输出进度推进后再标记“正在播放”；首块解码/排队中应显示“收到音频，缓冲中”。

## 本轮落地记录

- 角色级配置生成前重新拉取，避免同角色旧消息用过期音色配置覆盖新配置。
- 多音色默认三行：旁白、用户、角色；缺少映射时提示，不再静默使用第一个音色。
- 合成档位移出 LLM 设置，默认 `expressive` / 质量优先；单音色和多音色都带质量参数。
- 离线音频只在播放/生成闭环里保存，不在首页或历史恢复时批量同步。
- 后台媒体控制改为前后 10 秒；暂停后的 WebAudio 直播不再从头重播，等落盘后按记录位置续播。
- 歌词区固定高度并取消当前行字号变化，滚动加节流。
- 设置面板恢复底部抽屉式对齐，右上计数器和设置按钮统一尺寸。
